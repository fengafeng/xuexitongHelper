### V5 第二次修复 - 媒体检测不到 + 持续"默认音频"
- **问题根因1**: `_detectMediaType()` 在 `#iframe` 还没加载完成时运行（contentDocument=null），找不到任何媒体，默认 fallback 到 `'audio'`，走音频路径后持续循环检测
  - **修复**: contentDocument=null → 返回 `'unknown'`；`run()` 改为延迟检测，最多重试10次（每次2秒）
- **问题根因2**: `_detectMediaType` 末尾 `this.configs.mediaType='audio'` 硬编码兜底，导致即使没找到媒体也走音频逻辑
  - **修复**: 改为 `this.configs.mediaType='unknown'`，由调用者处理重试
- **改造点**:
  - `run()` 立即创建面板（UI 反应），然后启动 `_delayedMediaDetect()` 延迟检测
  - `_detectMediaType()` 新增 `#iframe` 内直接 `audio`/`video` 标签检测
  - `_playVideoAtIndex()` 不可用时重试3次（每次2秒）
  - 所有关键路径添加了详细日志输出

### V5 第三次修复 - 已完成检测导致视频不播放
- **问题**: `_detectTaskCompleted()` 检测到 `[aria-label="任务点已完成"]` 就立即返回 true，导致视频/音频根本不会播放，直接跳下一节。无论循环模式是否开启都跳过。
- **修复**: 所有 `_detectTaskCompleted()` 调用前添加 `!this.configs.loopMode &&` 条件判断
  - `_runContentPageAudio()`: `if (!this.configs.loopMode && this._detectTaskCompleted())`
  - `_runContentPageVideo()`: `if (!this.configs.loopMode && this._detectTaskCompleted())`
  - `_waitForNextSectionReady()`: `if (!this.configs.loopMode && this._detectTaskCompleted())`
- **效果**: 循环模式下忽略"已完成"标记，强制播放视频/音频；正常模式下保持原有行为（已完成则跳过）
